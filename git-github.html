Difference between Git & Github

Git in a nutshell is a tool to keep track of the changes made in a particular project. It provides the facility for a number of people to work on the same project simultaneously. This makes it really very easy for people to work together. There are few commands which makes it very easy for the user to keep their work updated.

Github on the other hand is a service provider which is used to store our repositories. It also provides a user interface for creating repositories. Recently, Git has also released a desktop app for easily managing the project state.

We will be seeing some of the most common processes used by anyone who uses Git and Github.

I will be using CLI (Comand Line Interface), these processes are much easier with the help of Github desktop version.

Creting a repositories(repo)

We can imagine a repository as a folder which contains important git files for tracking changes.

chnages made in the project will be in the Git track record.

  1. Create a folder and name it whatever you want your repo to be called.

  2. Open the folder and in that folder open the terminal.

  3. Type command "git init".

  4. Use command "git status" to check repos's status.

Staging the changes and making commit

Staging is the intermidiate step of commiting the changes made in the project.

Commiting means allowing git to keep the track of the changes made. After commiting all the chnages made in the project will be in the Git track record.

  1. For staging selective files use command"git add filename" and for staging all the file in the repo use git add.

  2. Use command 'git commit -m "Commit Message"'.

    3. Look at the images for the reference.



Checking previous commits

  1. To see the previous commits made use command"git log".

  2. Each commit have different codes called SHA which differentiate them.

    3. Look at the images for the reference.



Creating & Switching between branches

A Branch is a different line where people can work without effecting the main/master branch. By Default we are on main/master branch. When we are sure about the changes made in different branch other than main branch we can merge them with the main branch and those changes would reflect in the main branch.

  1. To create a new branch use command'git branch "branch name"'.

  2. To switch to the created branch use comand 'git checkout "branch name'.

  3. To check the existing branches use command "git branch"

    ,active branch would be marked with a star.
